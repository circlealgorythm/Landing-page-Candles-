# AGENTS

## Стек проекта

- `Next.js 14` на `app` router
- `React 18`
- `TypeScript` со `strict: true`
- `Tailwind CSS 3`
- `ESLint` через `next/core-web-vitals` и `next/typescript`

## Структура проекта

- `src/app` — корневой layout, глобальные стили и главная страница
- `src/components/layout` — каркас страницы
- `src/components/sections` — секции лендинга
- `src/components/ui` — переиспользуемые UI-компоненты
- `src/lib` — утилиты и контентные данные
- `src/styles` — дизайн-токены
- `public` — статические ассеты

## Требования к окружению

- `Node.js >= 18.17.0`
- `npm`

## Ключевые команды

```bash
npm install
npm run dev
npm run lint
npm run typecheck
npm run build
npm run start
```

## Правила кода

- Использовать `TypeScript` без `any`, если это не оправдано и явно не локализовано.
- Для импортов внутри проекта использовать alias `@/*`, настроенный в `tsconfig.json`.
- Следовать текущему стилю кода: двойные кавычки, точки с запятой, лаконичные функции и явные типы для props.
- Для props компонентов предпочитать `Readonly<{ ... }>` там, где это уже используется в проекте.
- Новые секции лендинга добавлять в `src/components/sections` и подключать через `src/components/sections/index.ts`, если это соответствует текущему паттерну.
- Переиспользуемые примитивы UI складывать в `src/components/ui`, а не дублировать разметку по секциям.
- Контент лендинга по возможности хранить централизованно в `src/lib/course-content.ts`, а не разносить строковые литералы по нескольким компонентам.
- Общие утилиты размещать в `src/lib`.
- Стили строить через `Tailwind CSS` и существующие токены из `src/styles/tokens.css`.
- Не ломать существующую семантику App Router: `src/app/layout.tsx` и `src/app/page.tsx` остаются точками входа.
- Метаданные сайта и базовые настройки обновлять через `src/lib/site-config.ts` и `metadata` в `src/app/layout.tsx`.

## Поведенческие правила агента

- Сначала прояснять задачу, а не молча додумывать детали: если есть неоднозначность, явно фиксировать допущения или запрашивать уточнение.
- При нескольких разумных трактовках не выбирать вариант скрытно: коротко обозначать варианты и риски, если это влияет на результат.
- Предпочитать простейшую реализацию, которая закрывает запрос: без лишней гибкости, преждевременных абстракций и функциональности сверх задачи.
- Если решение получается раздутым, упрощать его до минимального поддерживаемого объёма.
- Вносить только точечные изменения по запросу пользователя: не делать попутный рефакторинг, не переписывать соседний код и не менять стиль без необходимости.
- Удалять только те неиспользуемые сущности, которые стали лишними из-за текущих изменений; старый чужой dead code не трогать без отдельного запроса.
- Для нетривиальных задач сначала формулировать короткий план с проверками успеха, а затем доводить работу до верифицируемого результата.
- Для багфиксов, рефакторинга и рискованных изменений по возможности опираться на воспроизводимую проверку: `lint`, `typecheck`, `build`, локальный запуск или другие наблюдаемые критерии.
- Для тривиальных правок использовать здравый смысл и не замедлять работу избыточным процессом.

## Самопроверка перед завершением задачи

1. Запустить `npm run lint`.
2. Запустить `npm run typecheck`.
3. Запустить `npm run build`.
4. Если менялась UI-часть, поднять `npm run dev` и вручную проверить страницу на `http://localhost:3000`.
5. Проверить мобильную и десктопную версии, якорные ссылки, тексты и отсутствие визуальных артефактов.
6. Если менялись данные или SEO, отдельно проверить `metadata`, тексты CTA и контент в `src/lib/course-content.ts`.

## Что учитывать сейчас

- В проекте нет отдельного тестового раннера (`vitest`, `jest`, `playwright`) — основная автоматическая проверка сейчас это `lint`, `typecheck` и `build`.
- Отдельная конфигурация formatter-а не найдена, поэтому нужно придерживаться существующего стиля файлов.
- В репозитории уже есть `node_modules` и `.next`, но ориентироваться в работе следует на исходники из `src`.
- На текущем состоянии репозитория `npm run lint` проходит успешно.
- На текущем состоянии репозитория `npm run typecheck` падает из-за отсутствующих файлов в `.next/types`, которые включены в `tsconfig.json`. Если ошибка повторяется, сначала регенерировать Next-артефакты через `npm run dev` или `npm run build`, а при необходимости очистить stale-содержимое `.next`.
